The 30-Minute Cycle

Every 30 minutes, NEXUS wakes up and runs this loop:

Read ALL state — TASKBOARD, MESSAGES, each agent's STATUS, git status, tmux sessions. No truncation — everything.
Compress via Gemini — Sends the full state to Gemini 3.1 Pro (free, via Antigravity proxy). Gemini compresses it into a focused situation report.
Ask the Brain — Sends compressed report to Claude (the strategic brain call). Claude analyzes the situation and returns a JSON game plan: which agents to spawn, what tasks to assign, what commands to run.
Execute the plan — NEXUS parses the JSON and acts: spawns agents in tmux, runs shell commands, navigates browser, writes files. Each agent gets a specific prompt with their task.
Agents work independently — ALPHA/BRAVO/CHARLIE run in their own tmux sessions. They read their IDENTITY file, load prior context from brain-context cache, do their task, save findings back to cache, update STATUS, and exit.
Next cycle — 30 min later, NEXUS reads what happened and decides the next move.

How Agents Remember (Context Continuity)

Each agent session starts fresh — but they have three ways to maintain continuity:

1. Brain-Context Cache (long-term memory)

Agents call store_context(key, content) before exiting. Gemini compresses it to ~200 tokens. Next session calls get_summary(key) to load prior findings.

Persists forever. No expiration.

FREE via Antigravity proxy

2. Shared Files (coordination)

TASKBOARD.md, MESSAGES.md, STATUS.md, and research/ directory. Every agent reads these on startup. Any agent can write updates.

Persists forever. On disk.

FREE

3. Gemini Document Cache (within-session)

For large documents (PDFs, email dumps), agents call gemini-create-cache to upload once, then gemini-query-cache repeatedly without re-uploading.

Expires after 1-60 min. Session-scoped only.

FREE via Antigravity proxy

4. Session Resume (NEW)

Agent sessions are now tracked by ID. When NEXUS respawns an agent, it uses claude --resume {id} to continue with full conversation history intact.

Persists across sessions. Full context restored.

What Agents Can Do

Write Code & Files

Create, edit, and delete files anywhere in the project. Write reports, documents, research notes.

Run Commands

Git, npm, tests, builds, any shell command. Can install packages, run scripts.

Deep Web Research

Via gemini-deep-research — multi-step research with citations on any topic.

Analyze Documents

Via gemini-analyze-document — read and analyze PDFs, DOCX, CSV, any file format.

Browser Automation

SUPERVISE mode — screenshot, click, navigate via Chrome DevTools Protocol.

Coordinate

Post messages to MESSAGES.md, update TASKBOARD, assign tasks to each other.

Using This for a Legal Case

To use the pipeline for legal research (e.g., wrongful termination):

Drop your documents in .collab/case/ — emails, termination letter, contracts, performance reviews, anything relevant.
Update the TASKBOARD with legal tasks — "Research CA wrongful termination law", "Build timeline from emails", "Draft demand letter", etc.
Update IDENTITY files — Tell agents they're doing legal research, not coding. ALPHA reviews legal strategy, BRAVO does research, CHARLIE drafts documents.
Enable the driver — NEXUS starts the 30-min cycle. Agents pick up tasks, research with gemini-deep-research, cache findings in brain-context, write reports to research/.
Over 2-3 days — agents iteratively research, refine, and produce: legal memo, timeline, demand letter, EEOC charge narrative, comparable cases, strategy document.

Important Disclaimer

This system does research and document drafting. It is NOT a lawyer and cannot give legal advice. Use its output as a starting point for consultation with a real attorney. Always verify legal citations — AI can hallucinate case law.
